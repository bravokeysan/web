エグゼクティブサマリ

GitHub Issues 駆動運用の背骨は GitHub Copilot cloud agent。重い実装と repo 理解は Claude Code、comment-driven 委任と PR レビューは Codex、Antigravity は agent-first IDE の PoC 枠。
  • Issue → PR の標準自走運用は GitHub Copilot cloud agent が最も自然で、Issue 作成・更新・sub-issue・PR 作成まで GitHub 内で完結する。
  • Android Studio で Flutter を回す比率が高いなら Claude Code 優先、GitHub 上の comment-driven delegation と PR review を重視するなら Codex 優先。
  • GitHub Copilot は 2026-06-01 から usage-based billing へ移行予定のため、組織導入の費用感は前後で要見直し。
  • Google Antigravity は agent-first IDE として有望だが、GitHub Issues 駆動の中核には現時点では置きにくく、PoC・実験枠として扱う。
  • 料金・提供条件は変わりやすいため、資料更新時には必ず公式URLと checked_at を更新する。

ツール詳細

OpenAI Codex

comment-driven 実装委任・PR レビュー担当
GitHub Issue/PR コメントから @codex で cloud task を起動できる、comment-driven 委任と高シグナル PR レビューに強い実装エンジン。

得意なこと

GitHub Issue/PR コメントから @codex で cloud task 起動 AGENTS.md によるプロジェクト規約共有 GitHub 上でのコードレビュー（P0/P1 中心の高シグナル型） 自動レビュー 並列 cloud task 既存 codebase オンボーディング 複数ファイル編集とコマンド実行

GitHub Copilot / cloud agent

GitHub Issues 駆動運用の標準基盤
GitHub Issues の作成・割当・自走実装・PR 作成・コードレビュー・CLI 自動化を GitHub 内で完結できる、Issue 駆動運用の標準基盤。

得意なこと

GitHub Issue 作成 / 更新 / sub-issue 生成 Issue 割当による cloud agent の自走（repo 調査・計画・ブランチ・PR 作成） GitHub Copilot code review（any language） Copilot CLI + GitHub Actions による automation workflow repository custom instructions JetBrains / Android Studio / VS Code 互換 GitHub.com / IDE / CLI / Mobile / Actions まで surface が揃う

苦手なこと

Issue を割り当てた後に追加された Issue コメントは反映されない（PR コメント側で追記する運用が必要） code review は custom instruction files の先頭 4,000 文字までしか読まない GitHub docs は 2026-06-01 から request-based → usage-based billing へ移行と明記 1 タスクで複数リポジトリを横断しにくい Code review は人間の代替ではなく補助

Google Antigravity

agent-first IDE の PoC・実験枠
Agent Manager / Editor / Browser / Terminal / Source Control / Rules / Workflows / MCP を備えた、ローカルインストール型の agent-first 開発プラットフォーム。

得意なこと

Agent Manager（"Mission Control"）による複数エージェント管理 Editor / Browser / Terminal / Source Control の統合 Rules / Workflows（.agents/rules）による振る舞い制御 MCP によるローカルツール / DB / 外部サービス接続 Individual plan $0/月の無料枠 ローカルインストール（Mac / Windows / 一部 Linux）

苦手なこと

個人 Gmail 向け preview として案内されており、preview 色が強い 公式資料上、GitHub Issues を assignee にした PR 自走フローが明示的でない PR レビューは IDE 内 Review Changes 寄りで、外部レビュアーとしての役割は弱い 上位機能は Google AI Pro / Ultra と連動し、コスト管理が分かりにくい
