エグゼクティブサマリ

「Issue に仕様を書き、AI が自走して実装・PR・レビュー・テストを進める」運用には、GitHub Copilot family を運用の背骨に、Claude Code を実装エンジン、OpenAI Codex を comment-driven 委任とレビュー、Cursor を IDE 主導の代替候補として組み合わせる。
  • 運用の背骨は GitHub Copilot family。Issue 作成・更新・assign から cloud agent が repository research / plan / branch 変更 / PR 作成までを GitHub-native フローで進める。
  • 重い実装は Claude Code。codebase の深い読解、複数ファイル編集、Android Studio 公式サポートで Flutter 実装との相性が最も良い。
  • GitHub コメント駆動の委任と高シグナルレビューは OpenAI Codex。`@codex` で Issues/PRs から cloud task 起動、AGENTS.md による規約定着、P0/P1 集中レビュー。
  • Cursor は IDE 主戦場のチームに強い。clarifying questions → plan → background execution、subagents、Bugbot による PR レビュー。GitHub Issue assign → PR の標準線は Copilot ほど明示されていない。
  • 個人開発者は Copilot Pro が最短経路。Flutter 実装・既存 repo 理解・テスト補完を強めるなら Claude Pro を追加、GitHub delegation / review 重視なら Codex、IDE 主体なら Cursor Pro。
  • GitHub Copilot は 2026-06-01 から usage-based billing へ移行。組織導入の費用感は前後で要見直し。料金表だけでなく billing docs も併読のこと。

推奨運用フロー

Issue を GitHub の正本にして、実装エンジンを状況に応じて差し替える。
  1. GitHub Issue に仕様・受入条件を書く
  2. Copilot で Issue を整形・分割（仕様整理 / sub-issue 化）
  3. タスク種別で実装エンジンを選択
    • 標準タスク → Copilot cloud agent に assign
    • 重い修正・深い調査 → Claude Code に依頼
    • GitHub コメント駆動の委任 → Codex に `@codex`
    • IDE 主導 / 並列探索 → Cursor cloud agent / automations
  4. Draft PR が起こる
  5. PR review: Copilot / Codex / Bugbot / Claude Review
  6. 不足テスト補完: Claude Code / Copilot / Cursor
  7. merge
